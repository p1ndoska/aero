# Краткое описание автоматического тестирования

Автоматическое тестирование API является критически важным этапом разработки, обеспечивающим стабильность работы системы при внесении изменений. В представленном фрагменте кода используется библиотека **Supertest** для тестирования HTTP-запросов к API, а также **Jest** в качестве тестового фреймворка для backend-компонентов и **Vitest** для frontend-компонентов. Тесты охватывают три основные функциональные области: авторизацию пользователей через конечную точку `POST /api/login`, управление подкатегориями услуг через `POST /api/services-categories`, и генерацию защитных кодов (капчи) для форм обратной связи. Тестовый набор проверяет как авторизованный доступ к защищенным ресурсам (с использованием JWT-токенов и проверки ролей), так и обработку различных сценариев ошибок, включая валидацию входных данных, обработку несуществующих ресурсов и внутренних ошибок сервера. Все тесты используют мокирование внешних зависимостей (Prisma Client, bcrypt, jsonwebtoken) для обеспечения изоляции тестового окружения и независимости от реальной базы данных.
