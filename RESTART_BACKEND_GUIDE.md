# Инструкция по перезапуску backend

## Почему нужен перезапуск?

После добавления новых контроллеров и роутов, Node.js необходимо перезапустить, чтобы:
1. Загрузить новые файлы контроллеров
2. Обновить Prisma Client после миграций
3. Зарегистрировать новые роуты

## Шаги для перезапуска:

### Вариант 1: Если backend запущен в терминале

1. Остановите сервер: `Ctrl + C`
2. Перезапустите: `npm run server` (с nodemon) или `npm start`

### Вариант 2: Если backend запущен как служба

1. Найдите процесс Node.js и завершите его
2. Запустите заново

### Вариант 3: Автоматический перезапуск (рекомендуется)

В терминале backend:
```bash
cd backend
npm run server
```

Это запустит nodemon, который будет автоматически перезапускать сервер при изменениях.

## Проверка работы API

После перезапуска проверьте в браузере или Postman:

```
GET http://localhost:8000/api/vacancy-page-content
```

Должен вернуться JSON с контентом страницы вакансий.

## Если ошибка 404 все еще есть:

1. Проверьте, что сервер запущен на порту 8000
2. Проверьте консоль backend на наличие ошибок
3. Убедитесь, что миграция БД выполнена: `npx prisma migrate dev`
4. Проверьте, что Prisma Client обновлен: `npx prisma generate`
